
# Log file path <pathname> is invalid. (Error 3591)

 **Last modified:** December 30, 2015

 _ **Applies to:** Access 2013 | Access 2016_

Make sure the file is in the location specified in the  **Logfile** value of the **HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Office\14.0\Access Connectivity Engine\Transporter** key in the Microsoft Windows Registry or reconfigure the synchronizer on your computer.

To configure the Synchronizer


1. Start Replication Manager.
    
2. On the  **Tools** menu, click **Configure Microsoft Replication Manager**.
    
3. Follow the instructions in the wizard.
